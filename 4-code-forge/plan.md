# Code Forge Plan

Hier planst du deinen **Programmcode**. Du musst keine Profi-Begriffe kennen – schreibe einfach auf, was passieren soll.

## Was soll die Software tun?
- In 1–3 kurzen Saetzen:
	- Was passiert, wenn das Programm laeuft?
	- Was sieht/hoert/spuert der Benutzer?

Beispiele:
- „Wenn jemand den Knopf drueckt, geht das Licht fuer 5 Sekunden an.“
- „Ein Sensor misst die Temperatur und zeigt sie auf einem Display an.“

## Womit programmieren wir?
- Programmiersprache (z. B. Python, C/C++, Arduino, Scratch):
- Welches Geraet oder welche Plattform? (z. B. Mikrocontroller-Board, Laptop, Tablet):
- Brauchen wir besondere Bibliotheken oder Beispiele? (z. B. „Beispielcode von der Arduino-Website“)

## Unsere wichtigsten Funktionen
Schreibe die wichtigsten Dinge auf, die der Code koennen muss.

| Nummer | Was soll der Code machen? | Fertig? |
|--------|---------------------------|---------|
| 1 | | ja/nein |
| 2 | | ja/nein |
| 3 | | ja/nein |

## Testplan (pruefen, ob es klappt)
Ueberlege dir 2–5 einfache Tests.

| Test | Was pruefen wir? | Ergebnis |
|------|------------------|----------|
| 1 | | ok/nicht ok |
| 2 | | ok/nicht ok |
| 3 | | ok/nicht ok |

Schreibe bei „Ergebnis“ kurz dazu, was passiert ist (z. B. „LED bleibt aus“).

## Notizen und Ideen
- Was hat beim Programmieren gut funktioniert?
- Was war schwierig?
- Welche Ideen haettet ihr fuer eine naechste Version?

> Lege deinen Code direkt in diesem Ordner ab. Wenn du viele Dateien hast, kannst du Unterordner anlegen (z. B. `src/` fuer Code und `test/` fuer Testprogramme).
